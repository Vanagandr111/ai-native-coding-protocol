# Contributing

Keep the protocol small.

Good changes:

- clarify an existing rule
- add a concrete example
- remove ambiguity
- reduce agent failure modes
- keep the document easy to paste into context

Avoid:

- framework creep
- long theory sections
- tool-specific assumptions in the core protocol
- rules that force commits, installs, or irreversible actions without user consent

When adding a rule, prefer this shape:

1. failure mode
2. rule
3. short example
